AI Coding Sessions - Development Log

This document provides a summary of the key AI interactions, prompting strategies, and debugging workflows used to build the Dodge AI Context Graph.

Initial Prompting & Graph Modeling
Objective: Transform raw SAP JSONL data into a relational graph.
• Strategy: Prompted the AI to design a schema that captures the Order-to-Cash (O2C) flow.
• Iteration: Identified 7 key entities including Customer, SalesOrder, Delivery, Billing, JournalEntry, Payment, and Product.
• Implementation: Generated a processing script to transform raw data into a clean SQLite database with established relationships.

Debugging: SQLite Syntax Resolution
Issue: The system encountered errors when executing specific SQL queries generated by the LLM.
• Workflow: Isolated the cause to quote handling in SQLite, where single quotes are required for string literals.
• Resolution: Updated the system prompt to explicitly enforce single-quote usage for all SQL generation.
• Result: Stable and reliable natural language to SQL translation.

Iteration: Full-Bleed UI & Visual Parity
Objective: Ensure the interface matches the target design specifications and handles all screen resolutions.
• Workflow: Used browser-based verification to identify layout gaps and responsiveness issues.
• Implementation: Applied CSS refinements to ensure the graph component fills the available viewport.
• Visual Sync: Refined design tokens including highlighting, headers, and custom icons to ensure parity with reference materials.

Guardrails & System Integrity
The system implements structural integrity checks at the architectural level:
• Constraint: The LLM is restricted to a specific JSON schema for all responses.
• Verification: The backend performs validation on all LLM outputs before processing.
• Defense: Off-topic or unauthorized queries are automatically identified and rejected by the system guardrails.

Summary of Development Phases
• Research: Data schema analysis and relationship mapping.
• Execution: Core graph visualization and API development.
• Verification: Visual parity checks and layout stability testing.
• Delivery: Documentation and repository organization.
